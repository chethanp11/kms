# Application Purpose and Functionality

## Purpose
Describe what the application is for and what it must do. This is the place for copied-project-specific product documentation under `src/docs/`.

## What belongs here
- Application purpose
- Core functionalities
- User-facing workflows
- Functional boundaries
- Important product notes that support implementation

## Structure
- `purpose.md`: why the application exists and who it serves
- `functionalities.md`: the core behaviors and capabilities the application must provide

## Rules
- Keep this documentation aligned with `intent/` and `design/`.
- Do not duplicate detailed design artifacts here.
- Update this folder when the application purpose or functionality changes.
