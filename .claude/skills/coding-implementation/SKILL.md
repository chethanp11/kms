## Purpose
Implement scoped changes with maintainable code and minimal risk.

## When to use
- When changing source code.
- When navigating an unfamiliar codebase.
- When refactoring without changing behavior.

## Do
1. Make the smallest correct change.
2. Respect approved plan and design unless they require structure changes.
3. Keep code modular and testable.
4. Avoid hidden behavior.
