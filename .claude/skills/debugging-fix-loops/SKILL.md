## Purpose
Repair failures at the right layer without introducing side effects.

## When to use
- When validation or review finds a defect.
- When the failure source is unclear.
- When repeated rework suggests loop inefficiency.

## Do
1. Trace each failure to a root cause.
2. Fix the correct layer, not the symptom.
3. Keep repairs narrow.
4. Revalidate only what changed.
