name: review-quality-control
description: Review code and design for correctness, readability, completeness, and stage-gate readiness.
---

## Purpose
Catch drift and quality issues before validation or closeout.

## When to use
- When comparing code to design.
- When checking design completeness.
- When defining a quality gate between workflow stages.

## Do
1. Check for correctness and missing coverage.
2. Call out ambiguity directly.
3. Verify stage-gate readiness.
4. Separate review findings from implementation defects.
